# 2026-06-09 — bwoc-agent-installer: empty dir → v1.0.0 stable

Built the BWOC installer for non-coder end users from an empty directory to a stable
v1.0.0 release in one session. Two parts: a bilingual TUI setup wizard (`bwoc-setup`)
and thin bootstrap install scripts. Standalone repo `bemindlabs/bwoc-agent-installer`
(public), CI + release workflows publishing prebuilt binaries for 5 targets.

## What changed
- **`bwoc-setup`** — ratatui 0.30 / crossterm 0.29, std-only TUI wizard (same idiom as
  `bwoc-tui`). Three panes (options · explanation · key hints). Shells out to the `bwoc`
  CLI; **no `bwoc-*` crate dependency** (works against any installed `bwoc`). Stages:
  language → welcome → check bwoc → pick backend → [baseUrl] → workspace (path/mode/
  runtime/lang) → first agent (name/role/model/fallback) → advanced (opt-in) → verify → done.
- **i18n** (`src/i18n.rs`) — bilingual EN (default) + TH, `t(lang, en, th)` helper, first-stage
  language picker, **F2** live toggle, `--lang en|th` preset.
- **Design tokens** (`src/theme.rs`) — mirrors `bwoc_core::design`'s ANSI palette (ACCENT/
  TITLE/SELECTION/SUCCESS/WARNING/DANGER/MUTED + spacing + glyphs). Decoupled local copy.
- **Bootstrap** `scripts/install.sh` + `install.ps1` — detect OS/arch, download `bwoc`+
  `bwoc-agent` from BWOC-Framework releases + `bwoc-setup` from this repo's own releases,
  checksum-verify, install on PATH, launch the wizard. No Rust required.
- **CI/CD** — `release.yml` (build 5 targets, tar.gz/zip + sha256, on `v*` tags),
  `ci.yml` (fmt + clippy `-D warnings` + build/test on Linux/macOS/Windows).

## Decisions
- **Standalone, shell-out architecture** — the wizard orchestrates `bwoc` via captured
  subprocesses rather than linking `bwoc-core`, so it never goes stale against the CLI and
  stays a lean binary.
- **Decoupled release sources** — `bwoc` ships from BWOC-Framework; `bwoc-setup` from this
  repo's own releases (`download_*` helpers take a base-URL param).
- **English default** bilingual (user request) with one-keypress Thai.
- **Re-install in place** — detect an existing `bwoc` on PATH and install over its dir
  rather than dropping a second PATH-shadowed copy.

## Bugs surfaced and fixed
- **`curl | bash` launch** — wizard inherited the curl pipe as stdin → crossterm "Failed to
  initialize input reader". Fixed: `install.sh` execs `bwoc-setup </dev/tty`; binary adds
  stdin/stdout `is_terminal()` guards + a pre-flight `event::poll(0)` that prints guidance
  (bilingual) instead of the raw crossterm error.
- **F2 toggle re-ran action stages** — `toggle_lang()` called `enter_stage()`, which on
  RunInit/RunNew/Verify re-executed the `bwoc` command (double `bwoc new`). Fixed: skip
  action stages, preserve cursor/text buffer on re-localize.
- **`bwoc new` failed for every backend** — it requires `lintCmd/formatCmd/testCmd/buildCmd`
  and prompts for them on a TTY, but the wizard's captured subprocess can't prompt. Fixed:
  pass no-op `--lint-cmd/--format-cmd/--test-cmd/--build-cmd true`.
- **openai-compatible `--endpoint` / `--base-url`** — `bwoc new` has **no** baseUrl flag;
  both names were extracted from help *prose*, not real flags. Fixed: write `baseUrl` into
  the created agent's flat `config.manifest.json` (dep-free insert after the first `{`).
- **Workspace default** — was `~/bwoc-workspace`; changed to the current directory (user
  runs the installer inside the folder they want the workspace in).

## Gotchas (for future me)
- When checking `bwoc <cmd> --help` flags, grep only Options lines (`^\s{6,}--`) — the full
  help text includes prose that mentions `--flag`-shaped tokens that aren't real flags.
- **Dev-machine clobber:** re-install-in-place will overwrite the creator's cargo dev build
  at `~/.cargo/bin/bwoc`. On the dev box, test with `BWOC_BIN_DIR=$(mktemp -d)`.
- The TUI needs a real standalone terminal — it cannot run inside an embedded/IDE console or
  a command runner (no interactive TTY).
- macOS has no `timeout` (use `gtimeout`); the wizard refuses non-TTY cleanly (exit 2).
